# Tarot Atlas (навчальний проєкт для портфоліо)

Повноцінний застосунок про карти Таро українською мовою.

**Технології:**
- Сервер: Node.js (чистий HTTP) — без БД, усі дані в коді (початкові дані) або на JSON-файлі
- Клієнт: чистий JS (веб-компоненти), односторінковий застосунок із серверним відтворенням
- ШІ-тлумачення: OpenRouter / OpenAI (за бажанням)
- Сховище: localStorage (журнал розкладів) + JSON-файл на сервері (поширення)

## Можливості

- **Карта дня** — визначена карта на кожен день
- **Розклади:** 3 карти («Минуле·Теперішнє·Майбутнє»), Пентаграма балансу (5 карт), Кохання та стосунки (5), Кар'єра та гроші (5)
- **ШІ-тлумачення** — цілісний аналіз розкладу через LLM (OpenRouter/OpenAI), з трьома тонами: психологічний, містичний, практичний; якщо ключ API не налаштовано — працює тлумач на основі правил
- **Бібліотека карт** — 78 карт з ілюстраціями, значеннями й ключовими словами
- **Журнал розкладів** — збереження, нотатки, обране в localStorage
- **Профіль** — локальне ім'я (без реєстрації)
- **Публічні розклади** — поширення за позначкою + картка для соцмереж SVG (JSON-файл на сервері)
- **Аналітика** — PostHog (за бажанням, без ключа працює без неї)

## Структура

- `apps/api` — Node.js API (чистий HTTP, початкові дані, JSON-файл для поширення)
- `apps/web` — чистий JS, односторінковий застосунок (веб-компоненти)
- `infra` — docker-compose (більше не потрібен, залишено для сумісності)

## Швидкий старт

```bash
# 1. Встановити залежності
npm install

# 2. Запустити API
npm start:api

# 3. В іншому терміналі — клієнт
npm start:web
```

Або напряму:

```bash
node apps/api/server.js  # API на http://localhost:3000
node apps/web/server.js  # Клієнт на http://localhost:5173
```

Жодна БД не потрібна. API працює повністю автономно.

## Маршрути API

| Метод | Шлях | Опис |
|-------|------|------|
| GET | `/api/health` | Перевірка API |
| GET | `/api/tarot/cards` | Список карт |
| GET | `/api/tarot/spreads` | Визначення розкладів |
| GET | `/api/tarot/draw?count=3&type=classic3` | Випадковий розклад |
| POST | `/api/tarot/interpretation` | ШІ-тлумачення |
| GET | `/api/tarot/card-of-day` | Карта дня |
| POST | `/api/share/spreads` | Створити публічний розклад |
| GET | `/api/share/spreads/:slug` | Отримати публічний розклад |
| GET | `/api/share/spreads/:slug/social-card.svg` | Картка для соцмереж |

## ШІ-тлумачення

Підтримується OpenRouter та OpenAI-сумісні API. Налаштування в `apps/api/.env`:

```env
OPENROUTER_API_KEY=sk-or-...
LLM_API_URL=https://openrouter.ai/api/v1/chat/completions
LLM_MODEL=openai/gpt-4o-mini
```

Якщо ключ не вказано — застосунок використовує тлумачення на основі правил.

## Аналітика

Аналітика за бажанням через PostHog. Файл `apps/web/.env`:

```env
VITE_POSTHOG_KEY=phc_...
VITE_POSTHOG_HOST=https://eu.i.posthog.com
VITE_ANALYTICS_ENABLED=true
```

Для вимкнення: `localStorage['tarot-analytics-opt-out'] = 'true'`.
