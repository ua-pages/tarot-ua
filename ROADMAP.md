# 🛣️ Дорожна карта Tarot Atlas + Runes (майбутній проект)

## 📋 Структура розробки

Два паралельні проекти з共общим ядром тестування та архітектури:
- **Tarot Atlas** (поточний) — карти Таро 🃏
- **Runes** (майбутній) — рунічна система ⚔️

## 🎯 Етап 1: Фундамент (Modular Testing)

### 1.1 Модульне тестування без залежностей ⭐ [PRIORITY]

**Ціль:** Встановити архітектуру тестування для обох проектів

- [ ] **Налаштувати test runner** (Jest / Vitest без бандлерів)
  - Файл: `jest.config.js` або `vitest.config.ts`
  - Базова конфігурація для чистого JS/TS
  - Запуск: `npm run test`

- [ ] **Виділити чистий tarot-engine** (без залежностей на DOM/IndexedDB)
  - Переносити логіку з `js/services/tarot-engine` у чистий модуль
  - Тести для:
    - Перетасування карт
    - Логіка розкладів
    - Тлумачення карт (rule-based interpreter)
    - Датовий розрахунок "Карти дня"

- [ ] **Тести для констант** 
  - Карти (78 шт.)
  - Розклади та їхні шаблони
  - Ключові слова й значення

- [ ] **Тести для утиліт**
  - Валідація даних
  - Форматування тексту
  - Хеширование/генерація ID

**Очікуваний результат:** Core бібліотека `tarot-engine` з 80%+ покриттям, без залежностей на браузер

## 🔮 Етап 4: Runes проект (архітектура)

### 4.1 Планування
- [ ] Дизайн архітектури Runes (подібна до Tarot, але з рунічною логікою)
- [ ] 24 рени (Elder Futhark) — карти
- [ ] Розклади для рун (3-рун, 5-рун, Норнівський розклад тощо)
- [ ] Тлумачення (на базі того ж rule-based engine)

### 4.2 Переиспользование core
- [ ] Витягти загальну логіку у `shared/`:
  - `shared/interpretation-engine` — базовий движок тлумачення
  - `shared/spread-engine` — логіка розкладів
  - `shared/storage` — IndexedDB, localStorage обгортка
  - `shared/journal` — журнал розкладів (спільний для обох)

- [ ] Один набір тестів для shared модулів

### 4.3 Структура проекту
```
tarot-ua/
├── packages/
│   ├── shared/           # Загальна логіка (tarot + runes)
│   │   ├── src/
│   │   │   ├── interpretation-engine/
│   │   │   ├── spread-engine/
│   │   │   ├── storage/
│   │   │   └── utils/
│   │   ├── __tests__/
│   │   └── package.json
│   │
│   ├── tarot/            # Tarot-специфічна логіка
│   │   ├── src/
│   │   ├── __tests__/
│   │   └── package.json
│   │
│   └── runes/            # Runes-специфічна логіка
│       ├── src/
│       ├── __tests__/
│       └── package.json
│
└── apps/
    ├── web-tarot/        # Поточний Tarot Atlas
    ├── web-runes/        # Новий Runes застосунок (майбутній)
    └── shared-ui/        # Загальні компоненти UI
```
